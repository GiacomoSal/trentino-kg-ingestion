# Trentino KG Ingestion (Schema-Driven)

This repository contains the scripts for the ingestion and automated generation of a Knowledge Graph (KG) starting from OpenStreetMap (OSM) data. The system applies the **iTelos approach**, utilizing a modular pipeline to formally separate data extraction, schema-driven mapping, and entity unification (Reference Context and Personal Context).

## Repository Contents

### Core Pipeline (iTelos Layers)
To maximize data reusability and avoid redundant API calls, the monolithic execution has been split into three independent components:

* `1_extraction.py`: Handles the extraction of raw geographic data via OSMnx/osm2kg based on predefined filters. Saves the output locally as a serialized dataframe (`raw_osm_data.pkl`).
* `2_mapping.py`: Reads the raw data and dynamically applies the declarative rules to generate the Entity Graph (EG). It outputs an intermediate RDF file (`mapped_kg.nt`).
* `3_unification.py`: Performs Entity Unification by introducing the Personal Context (e.g., `app:Tourist`), creating logical relationships (e.g., `app:eatsAt`) with the Reference Context, and uploading the final unified graph to GraphDB.

### Configuration & Ontology
* `mapping.yaml`: Declarative configuration file that maps OSM tags to the target ontology classes (e.g., `etype:restaurant`).
* `teleontology.ttl`: Static file defining the class hierarchies for both the Reference and Personal Contexts.
* `teleology.ttl`: Static file containing the definition, domains, and ranges of Object and Datatype properties (e.g., `app:eatsAt`, `osm:name`).

### Generated Artifacts & Legacy
* `raw_osm_data.pkl`: Intermediate binary file containing raw OSM nodes (generated by script 1).
* `mapped_kg.nt`: Intermediate N-Triples file containing the mapped Entity Graph (generated by script 2).
* `old_script.py`: The legacy, monolithic generator (previously `generate_clean_kg.py`), kept for reference purposes but no longer actively used.

## Execution Flow

To successfully build and query the Knowledge Graph, follow these steps:

1. **Ontology Setup:** Before running the pipeline, manually import the `teleontology.ttl` and `teleology.ttl` files into your GraphDB repository. Ensure inference is enabled (e.g., RDFS-Plus).
2. **Extraction:** Run `python 1_extraction.py` to fetch data from OSM.
3. **Mapping:** Run `python 2_mapping.py` to translate raw data into RDF triples using `mapping.yaml`.
4. **Unification & Upload:** Run `python 3_unification.py` to link the contexts and upload the final data to the local GraphDB instance.
